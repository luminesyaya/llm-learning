# 训练目标：Next-Token-Prediction + CrossEntropyLoss

---

## 一、一句话概括

> 给定前文，预测下一个词；用交叉熵衡量预测和真实的差距；差距越小，模型越好。

---

## 二、从头到尾的完整流程

```
输入文本: "今天天气真好"
    ↓
[分词] Tokenization
    ↓
Token IDs: [12, 45, 23, 67, 8]
    ↓
[模型前向传播]
    ↓
Logits（每个位置对词表每个词的分数）
位置0: [0.1, 2.8, 0.3, ...]  → 预测位置1
位置1: [0.2, 0.1, 2.5, ...]  → 预测位置2
位置2: [0.3, 0.2, 0.1, ...]  → 预测位置3
位置3: [0.4, 0.3, 0.2, ...]  → 预测位置4
位置4: [...]  → 忽略（没有下一个词）
    ↓
[Shift 对齐]
    ↓
位置0的预测 → 对应真实词"天"（位置1）
位置1的预测 → 对应真实词"气"（位置2）
位置2的预测 → 对应真实词"真"（位置3）
位置3的预测 → 对应真实词"好"（位置4）
    ↓
[Softmax] 分数 → 概率
    ↓
[CrossEntropyLoss] 计算每个位置的损失
    ↓
所有位置求平均 → 最终损失
```

## 三、数学公式

### 条件概率

$$P(x_t \mid x_1, x_2, ..., x_{t-1})$$

### Softmax（分数→概率）

$$q_{t,i} = \frac{e^{z_{t,i}}}{\sum_{j=1}^{V} e^{z_{t,j}}}$$

### 单位置损失（CrossEntropyLoss）

$$\mathcal{L}_t = -\log(q_{t,c}) = -\log\left(\frac{e^{z_{t,c}}}{\sum_{j=1}^{V} e^{z_{t,j}}}\right)$$

数值稳定形式：

$$\mathcal{L}_t = -z_{t,c} + \log\left(\sum_{j=1}^{V} e^{z_{t,j}}\right)$$

### Shift 对齐

```
预测 = logits[:, :-1, :]
真实 = labels[:, 1:]
```

### 序列损失

$$\mathcal{L} = \frac{1}{T-1} \sum_{t=1}^{T-1} \mathcal{L}_t$$

### 批处理损失（完整形式）

$$\boxed{\mathcal{L} = -\frac{1}{B(T-1)} \sum_{b=1}^{B} \sum_{t=1}^{T-1} \log\left(\frac{e^{z_{b,t, x_{b,t+1}}}}{\sum_{j=1}^{V} e^{z_{b,t,j}}}\right)}$$

## 四、数值例子

词表 = ["猫", "狗", "鸟", "鱼", "兔"]（V=5），句子 = [0, 1, 2]

**模型输出 logits：**
- 位置0: z_0 = [2.0, 1.0, 0.1, 0.3, 0.5]
- 位置1: z_1 = [0.2, 0.1, 2.5, 0.8, 0.3]

**Softmax 后：**
- 位置0: q_0 = [0.52, 0.19, 0.08, 0.09, 0.12]
- 位置1: q_1 = [0.07, 0.06, 0.67, 0.12, 0.08]

**Shift 对齐：**
```
shift_logits = [z_0, z_1]       # 去掉位置2
shift_labels = [1, 2]           # "爱"(第1个词), "猫"(第2个词)
```

**计算损失：**
- L_0 = -log(0.19) = 1.66
- L_1 = -log(0.67) = 0.40
- L = (1.66 + 0.40) / 2 = **1.03**

## 五、代码实现（PyTorch）

```python
B, T, V = 2, 4, 5
input_ids = torch.tensor([[0, 1, 2, 3], [4, 0, 1, 2]])
labels = input_ids.clone()

logits = model(input_ids)  # (B, T, V)

# Shift
shift_logits = logits[:, :-1, :].reshape(-1, V)  # (B*(T-1), V)
shift_labels = labels[:, 1:].reshape(-1)          # (B*(T-1),)

loss = F.cross_entropy(shift_logits, shift_labels)
```

## 六、训练过程

```
损失 L 大
    ↓ 反向传播：计算梯度
梯度告诉每个参数：变大还是变小
    ↓ AdamW 更新参数
再次前向传播
    ↓ 损失 L 变小一点
重复 N 次
    ↓ 损失 L 很小
```

## 七、面试必背

**Q: 为什么用 CrossEntropyLoss？**

> 交叉熵损失是分类问题的标准损失，和最大似然估计等价。对于 next-token-prediction，要最大化正确词的概率，等价于最小化交叉熵。

**Q: 为什么需要 Shift？**

> 位置 t 的输入是 x_t，但要预测的是 x_{t+1}。Shift 把 logits 和 labels 错位对齐，让模型学会"根据前文预测下一个词"。

**Q: 损失函数的目标是什么？**

> 让每个位置对真实词的概率最大化，也就是让模型对下一个词的预测尽可能准确。

## 总结

| 问题 | 答案 |
|------|------|
| 训练目标是什么？ | 最小化 next-token-prediction 的交叉熵损失 |
| 数学公式？ | -1/(B(T-1)) * Σ Σ log(softmax(z)_{x_{t+1}}) |
| Shift 为什么？ | 位置 t 预测 t+1，需要错位对齐 |
| 损失越小代表什么？ | 模型预测越准确 |
