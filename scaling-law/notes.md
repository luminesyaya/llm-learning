# Scaling Law 面试速记卡（应用篇）

---

## 一、面试官最可能问的 6 个问题

### Q1：Scaling Law 在实际训练中有什么用？

> 主要用于两个地方：
> 1. **资源规划**：给定算力预算，决定模型该做多大、数据该喂多少（用 C = 6ND 和 Chinchilla 配比 D ≈ 20N 反算）
> 2. **超参数调优**：用小模型做网格搜索，拟合幂律关系，预测大模型的最优 batch size 和学习率，避免暴力调参

**加分回答**：

> 比如 Qwen2.5，先用 0.1B、0.5B、1B 等小模型搜最优超参数，然后拟合 α 和 β，直接外推到大模型。原来要跑几百组实验，现在只需要跑几组验证就行。

---

### Q2：怎么用 Scaling Law 找最优学习率和 batch size？

```
第一步：在小模型上做网格搜索
→ 选 0.1B、0.5B、1B、3B 几个小模型
→ 每个模型上搜 lr 和 batch size 组合
→ 记录每个组合的最终 loss、训练稳定性、收敛速度

第二步：确定每个规模的最优点
→ 综合三个指标：loss 最低 + 训练稳定 + 算力效率高

第三步：拟合幂律关系
→ B*(N) ∝ N^α，η*(N) ∝ N^β
→ log B* = α log N + c1
→ log η* = β log N + c2
→ 代入预测大模型的最优值

第四步：在大模型上做局部微调
→ 在预测值附近试 3-5 个点（lr 浮动 20-30%）
→ 选最稳定的配置
```

---

### Q3：batch size 和 learning rate 是什么关系？

> 两者不是独立的。batch size 变大时，梯度噪声减小，梯度估计更准，通常可以支持更大的学习率。实践中可以联合搜索（二维网格），也可以降维搜索（假设 η ∝ B^γ，沿曲线搜）。

| batch size | 梯度噪声 | 学习率 |
|-----------|---------|--------|
| 小（256） | 大 | 小，怕走歪 |
| 大（2048） | 小 | 大，敢走大步 |

典型关系：lr ∝ √(batch_size)，batch 翻 4 倍，lr 可以翻 2 倍。

---

### Q4：Dense 和 MoE 的超参数能共用吗？

> 不能。Dense 和 MoE 的参数组织方式、激活量、训练稳定性都不同，拟合出来的 α 和 β 指数往往也不一样。Qwen2.5 的做法是为两种架构分别建立最优配置映射，不能简单照搬。

```
Dense：所有参数每次都激活 → 梯度稳定 → lr 可以大一点
MoE：每次只激活部分专家 → 梯度稀疏 → lr 要调小一点，否则容易崩
```

---

### Q5：怎么理解"涌现能力"和 Scaling Law 的关系？

> 很多所谓的"涌现"（模型到某个规模突然会做某件事），从 Scaling Law 角度看，底层能力其实是平滑提升的。只是因为任务指标是离散的（比如过不过及格线 50 分），一旦从 48 分涨到 52 分，就表现为"突然会了"。实际是连续变化越过了一个阈值。

> 这也是为什么大家不只看下游 benchmark，而是同时看预训练 validation loss。Loss 持续下降说明模型在持续变好，下游任务过线只是时间问题。

---

### Q6：Scaling Law 有什么局限性？

1. **统计规律，不是精确公式** — 描述平均趋势，受数据质量、tokenizer、架构细节影响
2. **Loss 不是全部** — 预训练 loss 低不等于产品体验好，还取决于 instruction tuning、RLHF、tool use 等后训练
3. **数据质量比数量更关键** — 规模大了之后，单纯加低质量数据收益有限，数据清洗、去重、合成数据、domain mix 变得更重要

## 二、面试回答模板

> Scaling Law 在实际中主要用于两个场景：
>
> **第一，资源规划。** 给定算力预算，用 C = 6ND 和 Chinchilla 的结论 D ≈ 20N 反算出最优的参数量和数据量。现在的主流模型基本都是按这个原则设计的。
>
> **第二，超参数调优。** 先在小模型上做网格搜索，找到每个规模的最优 batch size 和学习率，然后拟合幂律关系，外推到大模型。Qwen2.5 就是用 0.1B 到 3B 的小模型做搜索，然后预测 7B/72B 的最优配置。原来要跑上百组实验，现在只需要几组验证，省了大量的时间和成本。
>
> 当然 Scaling Law 也有局限：它是统计规律，不是精确公式；预训练 loss 不等于产品体验；数据质量比数量更关键。

## 三、各模型实践速查

| 模型 | 做法 | 说明 |
|------|------|------|
| GPT-3 | 大参数 + 相对少数据 | Kaplan 路线，现已过时 |
| Chinchilla | 中等参数 + 4 倍数据 | 纠正了 GPT-3 的比例 |
| LLaMA | D≈20N | 严格遵循 Chinchilla |
| Qwen2.5 | 小模型搜超参 + 外推 | 用 Scaling Law 调优超参的典型 |
| DeepSeek-V3 | MoE + 大参数 + 大数据 | 结合 MoE 和 Scaling Law |

---

## 四、总结

| 面试官想问的 | 回答重点 |
|-------------|---------|
| 你懂 Scaling Law 吗？ | 幂律 + 边际递减 + Chinchilla 配比 |
| 你会用吗？ | 小模型搜超参 → 拟合幂律 → 外推大模型 |
| 你知道局限吗？ | 统计规律 + loss 不是全部 + 数据质量 |

> Scaling Law = 用小模型试路，用幂律外推，用最少实验找到大模型的最优配置。
